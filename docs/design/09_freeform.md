## 1. 개념 — 원형(archetype) × 정규 파라미터

- 모든 파츠는 **원형(archetype)** 에 속한다. M1 원형 = `body`·`barrel` 2종 (sight 는 M1 변형 없음).
- 원형이 **morph 파라미터 집합**을 정의하고, 개별 파츠(팝콘 코어 vs 불도그 코어)는 **기준 치수**만 다르다. morph 값은 기준 치수에 대한 **배율/오프셋**으로 작용한다.
- **정규화 원칙**: 저장·UI·undo 전부 `t ∈ [0,1]` 만 다룬다. `t = 0.5` = 그 파츠의 원래 모습(변형 없음). 실측 치수는 빌더가 `lerp(min, max, t)` 로만 계산한다.
  - 효과 1: 팝콘 코어와 불도그 코어가 같은 morph 값에서도 다른 실루엣 → 파츠 개성(02 §1)이 죽지 않는다.
  - 효과 2: 범위(min/max)를 나중에 재보정해도 **세이브는 그대로 유효**하다 — 마이그레이션 불필요(§6).
- **범위 보정 원칙 (개정)**: min/max 는 **설계 시점에** 토이 프로포션 봉투(§7)를 만족하도록 파츠별로 보정한다. 슬라이더 전 구간·전 조합이 항상 봉투 안이므로 **런타임 자동보정이 존재하지 않는다** — t 가 움직이면 형태가 반드시 움직인다(§5.0 불변 조항).

```ts
// src/game/morph.ts — 순수 데이터 + 순수 함수 (leaf, main.ts import 금지)
export type MorphArchetype = "body" | "barrel";          // M2+: "stock" | "grip"
export type MorphKey =
  | "bodyLength" | "bodyChub" | "bodyNose" | "bodyGripAngle"   // body
  | "barrelLength" | "barrelBore" | "barrelTip";               // barrel

/** 저장 단위 — 0..1 정규값. 키 없음 = 0.5 (기본형). 결정문 12 의 파츠 인스턴스별 morph */
export type MorphState = Partial<Record<MorphKey, number>>;

export interface MorphParamDef {
  key: MorphKey;
  archetype: MorphArchetype;
  labelKo: string;        // "길이" — 슬라이더 상단
  minLabelKo: string;     // "짧게"  — 슬라이더 왼끝 (초등 어휘, 3글자 이내)
  maxLabelKo: string;     // "길쭉하게" — 오른끝 (예외적 4글자 허용: 라벨이지 버튼이 아님)
  milestone: "M1" | "M1.5" | "M2";
  /**
   * t(0..1) → 스탯 델타. piecewise 두-기울기 (§4):
   *   t < 0.5 → deltaAt0 × (1 − t/0.5),  t ≥ 0.5 → deltaAt1 × ((t−0.5)/0.5)
   * t=0.5 에서 전부 0 (기본형 = 카탈로그 표 수치 그대로)
   */
  statDelta(t: number): Partial<{ power: number; fireRate: number; accuracy: number; weight: number }>;
}
export function resolveMorph(state: MorphState, key: MorphKey): number; // 없으면 0.5, NaN/범위 밖은 clamp
```

## 2. 파라미터 정본 표

범위는 **기준 치수 대비 배율**(길이류) 또는 **형태 계수**(모양류). 실측 기준치는 03 §1.2 레시피가 소유. **모든 범위는 §7 봉투 검산 교차표를 통과하도록 사전 보정된 값이다** — 초안 대비 축소분(길이 ×1.40→×1.25 등)은 전부 봉투 위반을 설계 시점에 제거하기 위함.

### 2.1 몸통 (body) — M1 3개 + M2 1개

| key | 라벨 (왼끝↔오른끝) | 범위 | M | 지오메트리 효과 |
|---|---|---|---|---|
| `bodyLength` | 길이 (짧게↔길쭉하게) | 길이 ×0.75~×1.25 | **M1** | 셸 RoundedBox Z 치수 재생성. 캐리핸들·파팅라인 띠도 비례 추종 |
| `bodyChub` | 통통함 (홀쭉↔통통) | 단면(X·Y) ×0.85~×1.35 | **M1** | 셸 단면 재생성 + 라운드 반경 재계산(§7 — 스케일로 뭉개지 않는다) |
| `bodyNose` | 코 모양 (뭉툭↔쫑긋) | 돌출·라운드 계수 0~1 | **M1** | **단일 패밀리 연속 정의**: 캡은 항상 Cone+Sphere. t 가 [돌출 길이 0→max]와 [캡 라운드 반경 max→min]을 **동시에 lerp** — t=0 은 돌출 0 = 사실상 반구(뭉툭), t=1 은 길고 뾰족-둥근 코(쫑긋). 토폴로지 스왑 없음 → 중앙 자석 스냅(§5)과 충돌하지 않는다. **스탯 0 — 순수 멋** |
| `bodyGripAngle` | 손잡이 기울기 (꼿꼿↔비스듬) | −5°~+25° | M2 | 그립 소켓 앵커 회전 + 방아쇠울 위치 추종 |

### 2.2 배럴 (barrel) — M1 2개 + M2 1개

| key | 라벨 | 범위 | M | 지오메트리 효과 |
|---|---|---|---|---|
| `barrelLength` | 길이 (짧게↔길쭉하게) | 길이 ×0.70~×1.25 | **M1** | 튜브 재생성 + **muzzle 앵커 전진**(§3.3) + 줄무늬 재도색(§3.4) |
| `barrelBore` | 굵기 (가늘게↔두툼하게) | 반경 ×0.90~×1.55 | **M1** | 튜브·머즐링 반경 재생성. 발사체 크기는 **04 ShotProfile 의 `projectileScale`(시각·충돌 공용, 04 소관 신설 요청) 입력에 기여**한다 — 09 는 기여치 제안만 소유(`boreScale = lerp(0.9, 1.3, t)`), 파워→크기 채널과의 합성식·충돌 반경 동기화는 04 정본에 위임. 시각 전용 크기 채널은 두지 않는다(보이는 대로 맞는다) |
| `barrelTip` | 끝 모양 (밋밋↔나팔) | 끝단 플레어 0~1 | M2 | 끝에 Cone 나팔 성장. 스탯 0, 발사 파티클 퍼짐 연출만 |

- **코 모양·끝 모양처럼 스탯 0 파라미터를 반드시 섞는다** — "성능 때문이 아니라 그냥 예뻐서" 바꾸는 자유가 조형 놀이의 절반이다.
- sight 는 M1 변형 없음(작은 파츠라 체감 낮음). M2 에 `scope 원형` 검토(§8).

## 3. 지오메트리 전략

### 3.1 빌더 규약 — buildPart 시그니처 확장 (03 §3.1 개정)

```ts
export interface BuildOpts { morph: MorphState; lod?: "drag" | "full"; } // 기본 "full"
export interface BuiltPart {
  group: THREE.Group;
  zones: Partial<Record<ZoneId, THREE.Mesh[]>>;              // 03 계약 유지
  anchors: Partial<Record<SocketId, THREE.Object3D>>;        // ★신설 — morph 반영된 소켓 앵커
  dispose(): void;   // 인스턴스 지오메트리만 정리. 머티리얼은 캐시 소유 — 절대 dispose 금지(03 §5)
}
export function buildPart(partId: PartId, opts: BuildOpts): BuiltPart;
```

- **03 §5 의 "같은 partId 는 base geometry 공유" 규칙은 morph 도입으로 개정한다**: morph 대상 파츠의 지오메트리는 **인스턴스 소유**이며 재생성 시 이전 것을 즉시 dispose. 비대상 파츠(sight)만 종전 공유 유지.
- 빌더는 순수 함수: 같은 `(partId, morph)` → 같은 형태. 사격장 견착 뷰·썸네일·공방이 **전부 이 하나를 공유**하므로 "공방에서 늘인 총이 사격장에서 원래대로" 같은 배신이 구조적으로 불가능하다.
- 03 §1.1 규약(길이축=Z 굽기·원점 센터·전방 -Z)은 morph 빌드에도 그대로 적용.

### 3.2 재생성 스로틀 + LOD

| 국면 | 주기 | LOD | 내용 |
|---|---|---|---|
| 슬라이더 드래그 중 | **33ms(≈30Hz) 스로틀** | `drag` | 변형 중인 파츠 1개만 재빌드. 세그먼트 절반(RoundedBox seg 4→2, Cylinder radial 12→8) |
| pointerup | 즉시 1회 | `full` | 풀 세그먼트 재빌드 + 줄무늬 재도색 + undo push + 자동저장 flush |
| 프리셋 시드 버튼 (M1.5) | 즉시 1회 | `full` | 동일 (드래그 국면 없음) |

- 비용 상한: 파츠당 메시 ≤14(결정문 23), full LOD ≤2,500 vert·drag LOD ≤1,200 vert → 재빌드 1회 <1ms 급. 30Hz 스로틀은 안전 여유다.
- 재빌드는 **pointer 이벤트 경로에서만** 발생 — `update*/animate*/tick*` 밖이므로 07 핫패스 할당 게이트와 충돌 없음(빌더 이름도 `rebuild*` 로, `update*` 금지).

### 3.3 소켓 앵커 재계산 — "형태가 늘면 소켓이 따라간다"

- **몸통 빌더**가 자기 소켓 앵커를 morph 반영 좌표로 반환한다: `bodyLength` t=1 이면 muzzle 소켓 z 가 전진, mag·grip 앵커는 비례 위치(비율 보존) 유지.
- **배럴 빌더**는 `muzzle` 앵커(총구 끝)를 반환한다 — M2 총구 액세서리는 몸통이 아니라 **배럴 끝 앵커에 승계 부착**된다. `barrelLength` 를 늘이면 나팔·안전팁이 같이 전진하는 게 이 계약의 존재 이유다.
- **부착 방식 (개정)**: 자식 파츠는 **앵커의 자식으로 두고 로컬 원점에 고정**한다 — `anchor.add(child); child.position.set(0,0,0)` (03 §1.1 부착 규약). 앵커가 전진하면 자식이 자동으로 따라온다. **`Object3D.attach()` 사용 금지** — attach() 는 재부모화 시 월드 변환을 보존하는 시맨틱이라 앵커가 전진해도 자식이 제자리에 남는다(의도와 정반대). verify 에 단위테스트 1개 고정: `barrelLength` t=1 시 muzzle 자식의 월드 z 전진량 = 앵커 전진량.
- 조립 합성(`blasterVisuals`)의 재부착 순서: 파츠 재빌드 → 앵커 diff → 자식 파츠 재부착(위 방식) → 존 페인트 재적용(§3.4). 이 순서를 함수 하나(`recomposeBlaster`)로 고정해 호출처가 순서를 틀릴 수 없게 한다.

### 3.4 페인트·줄무늬와의 순서 계약

색|finish 머티리얼 캐시(결정문 10, 03 §5)는 지오메트리와 무관해서 재빌드 후 **재할당만** 하면 된다. 줄무늬(M2, vertex color)는 boundingBox 기반이므로 **재빌드 때마다 재도색** — `recomposeBlaster` 마지막 단계에 고정.

### 3.5 성능 예산 준수 (07 단일표 참조)

morph 는 **메시 수·머티리얼 수·draw call 을 바꾸지 않는다** (지오메트리 attribute 만 교체). 블래스터 draw call ≤56·파츠당 메시 ≤14 은 morph 극단값에서도 불변 — verify 테스트가 극단 morph 로 `countMeshes` 를 재검사한다(§7).

## 4. 스탯 결합 — "형태가 성능이다"

원칙: **morph 스탯 델타는 파츠 교체 델타(02 §3)보다 한 급 작게** — 파츠 선택이 큰 방향, morph 는 미세 조정. 최종 스탯은 02 clamp(1~10), 발사 변환은 04 ShotProfile 클램프(탄속 20~60m/s·퍼짐 0.3~4.0°·간격 150~600ms, 결정문 4) 안이므로 morph 가 계약을 뚫을 수 없다.

**델타 모델 (개정 — piecewise 두-기울기로 확정)**: 파라미터마다 [t=0 델타]와 [t=1 델타]를 독립 정의하고, `t<0.5` 구간은 t=0 값→0 으로, `t≥0.5` 구간은 0→t=1 값으로 선형 보간한다. 왼끝·오른끝 델타가 대칭일 필요가 없다(비대칭이 이 모델의 존재 이유). **표의 빈 칸 = 0 이 의도다.** `statDelta` 단위테스트에 파라미터별 t=0/0.5/1 세 점을 고정 스냅샷.

| key | t=0 델타 (왼끝) | t=1 델타 (오른끝) | 직관 |
|---|---|---|---|
| `bodyLength` | 연사 +0.5 / 정확 −1.0 | 연사 −0.5 / 정확 +1.0 | 긴 몸 = 안정, 짧은 몸 = 잽싸다 |
| `bodyChub` | 파워 −1.5 / 무게 −1.5 / 연사 0 | 파워 +1.5 / 무게 +1.5 / **연사 −1.0** | 통통 = 세지만 무겁다(무게↑ → handling↓ 자동 파생) |
| `bodyNose` | 0 | 0 | 순수 멋 |
| `barrelLength` | 정확 −1.5 / 무게 −1.0 / 연사 0 | 정확 +1.5 / 무게 +1.0 / **연사 −0.5** | 긴 배럴 = 명사수, 대신 둔해짐 |
| `barrelBore` | 파워 −1.0 / 정확 0 | 파워 +1.0 / **정확 −0.5** | 두툼한 다트 = 세지만 살짝 퍼짐 |

- `computeStats` 를 **morph 인지형으로 확장**한다: `몸통 base + Σ파츠 delta + ΣmorphDelta(인스턴스별)` 후 clamp — 호출부는 기존과 동일하게 이 함수 하나만 본다(02 §5 원칙 유지).
- **성능 체감 채널 3종 (개정 — 별 반 개 깜빡임만으로는 체감 0)**:
  1. **연속 별 채움**: 스탯 별을 부분 채움(연속)으로 렌더 — 드래그 중 별이 미세하게 차오르고 빠진다. 숫자 없음(결정문 3)은 유지.
  2. **퍼짐 원 프리뷰**: 슬라이더 패널에 정확 스탯 연동 미니 레티클(원 지름 = 04 퍼짐각 매핑 비례)을 상시 표시 — 배럴을 늘이면 원이 조여지는 게 별보다 즉물적. computeStats→ShotProfile 동일 경로를 쓰므로 거짓 프리뷰가 불가능.
  3. **사격장 바로가기**: 슬라이더 패널에 "쏴보러 가기" 버튼 1개 — 기존 스테이션 전환(06 StationId) 재사용으로 풍선 마당(결정문 20) 직행. 공방 내 1발 시사는 M2 검토(기각한 비평 참조).
- **± 팝업 (개정 — 숫자 금지)**: 0.5별 경계를 넘는 pointerup 에 **별 단위 아이콘 팝업**(`정확 ▲` + 반 별 글리프) 발사. `+1 정확` 같은 숫자 표기는 결정문 3 위반이라 폐기. **02 §4.1 의 파츠 교체 팝업도 동일 위반이므로 별 단위 표기로 개정 필요 — 02 소관으로 플래그.**
- 발견 재미의 정점: **같은 파츠 5종으로도 morph 조합이 다르면 다른 총이다.** "롱 레일을 최대로 늘인 홀쭉 팝콘" vs "숏 스냅을 두툼하게 만든 통통 불도그" — 카탈로그 밖의 내 총.

## 5. 조작 UX

### 5.0 UX 불변 조항 (신설)

> **슬라이더 t 가 움직이면 화면의 형태가 반드시 움직인다.** 죽은 구간·침묵 자동보정·교차 커플 압축 금지. 이를 보장하는 수단이 §7 의 설계 시점 범위 보정(봉투 사전 검산)이다 — 런타임에 슬라이더 값과 형태가 어긋나는 경로 자체를 없앤다.

- **진입**: 공방(workshop)에서 파츠 탭 → 우측 패널(결정문 22 가로 레이아웃)에 그 파츠의 **큰 슬라이더 2~3개**가 카탈로그 아래 슬라이드-인. 별도 "변형 모드" 없음 — 조립과 변형은 같은 공방의 한 호흡.
- **슬라이더 사양**: 높이 56px·썸 44px(06 §10), 양끝에 초등 어휘 라벨 + 절차 SVG 아이콘(결정문 25, 이모지 금지). 중앙(0.5)에 틱 + ±0.03 자석 스냅 — "원래대로"가 손끝으로 찾아진다. 수치 표기 없음. (nose 의 단일 패밀리화로 중앙 스냅 부근 형태 지터 문제는 소멸 — §2.1)
- **실시간 반영**: 드래그 중 30Hz 재빌드(§3.2)로 총이 손을 따라 늘어난다 — 이 즉물감이 기능의 전부다. 변형 중인 파츠는 0.5s 존 펄스 1회(03 §6 재사용)로 "지금 이걸 주무르는 중" 표시.
- **카메라**: 파츠 선택 시 해당 소켓으로 15% lerp + 0.9× 줌(03 §6 그대로), 슬라이더 드래그 시작 시 자동회전 정지·종료 시 재개(색칠 모드와 동일 규칙).
- **프리셋 시드 버튼 3개 (M1.5 — §8)**: `날렵하게`(길이↑·통통함↓·코 쫑긋) / `우람하게`(통통함↑·굵기↑·길이 약간↑) / `랜덤 섞기`(전 파라미터 t ∈ [0.1, 0.9] 균등 샘플). 각 버튼 = full 재빌드 1회 + undo 1엔트리. 랜덤은 봉투 사전 보정(§7) 덕에 어떤 결과도 안전하다 — "랜덤 누르고 웃음 터지기"가 공식 놀이. **M1.5 첫 항목 = 랜덤 섞기** (재미 밀도 최고, 유실 방지 명기).
- **사운드 (M1.5)**: 드래그 중 80ms 스로틀로 "뽀롱" 피치가 t 를 따라 슬라이드(audioKit sine 500+600·t Hz — 밝고 높게, 전작 교훈), pointerup 에 "딸깍".
- **되돌리기 관계 (결정문 15, 개정 — paint 미캡처)**: morph 는 **조립(assembly) 스택과 통합** — M1 되돌리기 버튼 1개가 파츠 교체·변형을 모두 되감는다. **1 슬라이더 제스처 = 1 undo 엔트리** (pointerdown 에 이전 상태 캡처, pointerup 에 push — 드래그 중 30Hz 재빌드가 30개 엔트리가 되지 않게).
  - **스냅샷 범위**: `{slot: {partId, morph}}` 로 한정한다. **조립 스택은 paint 필드를 절대 캡처하지 않는다** — 복원은 현재 paint 를 보존하는 merge 로 수행하고, 스냅샷과 partId 가 달라져 paint 대상이 사라지는 슬롯만 기본색 폴백. (Blaster 전체를 캡처하면 paint 스테이션에서 칠한 색이 공방 되돌리기로 몰래 롤백돼 결정문 15 의 kind 분리 취지가 깨진다.)
  - 스냅샷은 수백 바이트라 30깊이 무부담.
- **자동저장** (결정문 14): pointerup·프리셋 버튼이 06 의 "변형 직후" 트리거로 flush.

## 6. 직렬화 · 마이그레이션

결정문 12·13 정합 — `SAVE_VERSION 1` 부터 morph 포함, 파츠 인스턴스 단위 저장:

```ts
export interface PartInstance {
  partId: PartId;
  paint: PartPaint;      // 3존 PaletteKey — 03/08 계약
  morph: MorphState;     // 0..1 정규값. 기본값(0.5) 키는 저장 생략 (희소 Record)
}
export interface Blaster {
  id: string; name: string; createdAt: number;
  parts: Partial<Record<SlotType, PartInstance>>;   // body 만 필수 (결정문 16)
}
```

로드 규칙 (전부 무소음 복구 — 아이에게 에러 다이얼로그 금지):

| 상황 | 처리 |
|---|---|
| 모르는 MorphKey | **조용히 무시** + 콘솔 경고 (미래 버전 세이브의 하위 호환) |
| 키 개명 | 현재 사례 0건. 최초 개명 커밋에서 `MORPH_KEY_RENAMES: Record<string, MorphKey>`와 마이그레이션 테스트를 함께 추가 |
| 파라미터 삭제 | 키 무시 → 해당 축은 기본형(0.5)으로. 형태만 살짝 돌아올 뿐 세이브는 산다 |
| 값이 NaN·범위 밖 | 0..1 clamp, NaN 은 0.5 |
| 범위(min/max) 재보정 | **마이그레이션 불필요** — 정규값 재해석이 정규화 저장의 존재 이유 |

- roundtrip 테스트(07)에 "morph 포함 저장→로드→deepEqual" + "미지 MorphKey 세이브 로드" 케이스 추가.
- 파라미터 **추가**는 언제나 안전(없는 키 = 0.5). SAVE_VERSION 증가가 필요한 경우는 MorphState 의 **구조** 자체가 바뀔 때뿐.

## 7. 토이 프로포션 봉투 — 설계 시점 사전 보정 (전면 개정)

자유 범위의 바깥 울타리. 런타임 자동보정은 두지 않고, 렌더러와 테스트가 같은 실측 함수(`bodyShellMetrics`·`barrelTubeMetrics`)를 사용해 범위 위반을 개발 시점에 차단한다.

1. **파츠별 실효 min/max 를 설계 시점에 사전 계산** — 각 partId의 `lerp(min,max,t)` 전 구간·전 조합이 봉투 안이어야 한다.
2. **76종 로스터 재기준선(2026-07-22)** — M1의 5종 기준 수치는 장척·소형 토이 계열 확장 후 실제 카탈로그와 불일치했다. 현재 상한 ratchet은 몸통 세장비 **6**, 배럴 L/r **28**, 장착부 최소 반경 **0.019m**, 박스 모서리 최소 **8%**(기본 **25%**), 몸통+배럴 전장 **1.30m**다. 배럴 앞끝은 테이퍼 표현을 위해 별도 **0.012m** 하한과 안전 끝장식으로 보호한다.
3. **런타임 클램프는 assertion 으로 강등** — 위반은 자동보정이 아니라 **테스트 실패**(개발 중 발견)다. 배포 빌드에서 봉투 위반 경로는 존재하지 않는다.
4. **전장 쿼터는 정적 예산 분할로 해소** — 전 몸통×배럴 최대 길이 조합이 **1.30m**를 넘지 않는지 전수 검증한다.

### 7.1 봉투 검산 교차표 (03 §1.2 기준 치수 대조 — 최대 파츠 기준 worst-case)

기준: 76종 전체 카탈로그의 실측 함수 결과. 아래 값은 현재 worst-case이며 상수는 소수 오차와 신규 파츠 검토 여유를 둔 ratchet이다.

| 검산 | worst 조합 | 값 | 봉투 | 판정 |
|---|---|---|---|---|
| 몸통 세장비 | 로켓 몸통 길이 max / 단면 min | 5.89 | ≤ 6 | 통과 |
| 배럴 L/r | 니들 배럴 길이 max / 장착부 반경 min | 27.78 | ≤ 28 | 통과 |
| 최소 장착부 반경 | 니들 배럴 bore min | 0.0198m | ≥ 0.019m | 통과 |
| 합성 전장 | 타이탄 몸통 max + 니들 배럴 max | 1.275m | ≤ 1.30m | 통과 |

- `tests/envelope.test.ts`가 전 partId × 범위 꼭짓점과 partId별 결정적 랜덤 50콤보, 전 몸통×배럴 최대 전장을 검산한다. 기준 치수나 morph 범위가 바뀌면 즉시 실패한다.

## 기각한 비평

10건 중 전면 기각은 없음. 2건은 **문제 제기는 수용하되 제시된 수정안 대신 다른 메커니즘을 채택**했다:

- **비평 2 의 수정안 중 "배럴 실효 범위를 현재 몸통 길이에서 매번 재산출 + 트랙에 사용 가능 영역 표시" 부분 기각** — 동적 범위는 죽은 구간은 없애지만 새 혼란을 만든다: 몸통 슬라이더를 움직이면 배럴 슬라이더의 의미(같은 t = 다른 길이)가 바뀌고, 극단에선 몸통을 늘리는데 배럴이 눈앞에서 줄어든다. 초등학생에게 이 교차 인과도 "안 먹네"와 같은 급의 배신이다. 대신 **정적 예산 분할**(몸통 max + 배럴 max ≤ 쿼터, §7)로 커플링 자체를 제거 — 비평이 요구한 원칙("t 가 움직이면 형태가 반드시 움직인다", §5.0)은 더 강한 형태로 수용했다.
- **비평 5-③ 의 "공방 내 1발 시사" 부분 기각(M2 이월)** — 공방 씬에 발사·탄도·충돌을 이식하는 것은 사실상 미니 사격장 하나를 더 만드는 일로, 축소된 M1 예산(비평 9 수용)과 정면 충돌한다. 대신 **사격장 바로가기 버튼**(기존 스테이션 전환 재사용, 구현 수 분)으로 "바꾸고 바로 쏴본다" 루프는 M1 에 성립시키고, 공방 내 시사는 M2 검토 항목으로 남긴다.
