# 연령 적합성 · 아트 디렉션 가드레일

> **이 섹션의 지위**: 하드 룰. 다른 모든 섹션(파츠 시스템, 타겟/사격장, 이펙트, UI 텍스트, 자유 변형)은 본 섹션의 제약을 그대로 상속한다. 위반은 "취향 차이"가 아니라 **버그**로 취급하고, §3의 기계 점검(vocab lint)과 §5의 릴리스 체크리스트로 잡는다. **금칙어 리스트의 단일 정본은 본 §3.1 이다** — 03 에 있던 별도 목록은 본 표로 병합되었고 03 은 참조만 한다(결정문 25).
>
> **개정 요지 (비평 반영)**: ① vocab lint 를 오탐 없는 기계 명세로 재정의(표=정본, 토큰 단위 매칭, allowlist), ② 타겟 눈·표정 기본 금지(경계 침식 차단), ③ 시뮬레이션 피드백은 정직한 수치+트레이드오프(긍정 프레임은 말투만), ④ paintSplat 은 M2+ 강등 + 구현 방식 확정, ⑤ 측정 불가한 "라운드 비중 ≥50%" 폐기 → 성능 예산으로 교체, ⑥ 색은 hex 가 아니라 `PaletteKey` 로 저장, ⑦ 다크 톤은 존 차등 허용, ⑧ SFX 캡 2단화, ⑨ §5·§6 통합 + 트리거를 실존 프로세스(마일스톤)로 교체.
>
> **결정문 반영 (2차 개정)**: 발사체 kind = 04 정본 3종으로 정리·suctionArrow 삭제(결정문 6), 성능 수치 예산은 07 단일표 참조화(결정문 23), 수치 노출을 결과 화면 기록값으로 한정(결정문 3), 색 존을 paint 3존(primary/secondary/accent)에 정합 + PaletteKey 폴백 규칙 신설(결정문 11·12), 소리 파츠 재정의·SVG 아이콘·"진열장" 어휘(결정문 25), 오류 화면 톤 규칙(결정문 26), 자유 변형(09) 가드레일 연결 신설(결정문 27 — **상세 정본은 09**).

## 1. 토이 블래스터 아트 디렉션

### 1.1 형태 언어 — "둥글고 통통하고 과장되게"

핵심 판별식: **실루엣만 봤을 때 장난감 가게 진열대에 있어도 어색하지 않은가?** 실총의 실루엣(얇고 길고 각진)에서 멀어질수록 안전하다. 이 판별식 자체는 수치 규칙이 아니라 **§5 스크린샷 체크리스트 항목**으로 검증한다(측정 불가능한 "비중 %" 규칙은 두지 않는다).

| 항목 | 규칙 | 수치 가이드 |
|---|---|---|
| 프로포션 | 실총 대비 몸통을 뚱뚱하게 과장 | 몸통 높이:길이 비율 ≥ 1:2.5 (실총은 1:5~1:7) — 짧고 통통하게 |
| 모서리 | 각진 엣지 금지, 라운드 프리미티브(`CylinderGeometry`·`CapsuleGeometry`·`SphereGeometry`)를 적극 사용. 박스는 겹쳐 쌓아 베벨 느낌으로 각을 죽인다 | "둥근가" 판정은 §5 실루엣 체크리스트로. 코드 규칙은 아래 지오메트리 규칙 2개뿐 |
| 총구 | 뾰족한 총구 금지 | 총구 끝은 항상 **깔때기/나팔 모양**(반지름이 몸통보다 커지는 플레어) 또는 둥근 캡. 다트 끝이 살짝 보이면 더 장난감답다 |
| 디테일 | 리얼한 기계 디테일(레일·볼트·강선) 금지 | 대신: 큼직한 나사 머리(장난감 조립 느낌), 투명 창(내부 다트 보임), 스프링 노출 장식 |
| 스케일 과장 | 파츠는 실기능 대비 1.5~2배 크게 | 조준경은 왕눈이처럼 크게, 개머리판은 쿠션처럼 두툼하게 — "가챠 장난감" 감각 |

**지오메트리 규칙 (하드 — 검증 가능한 것만)**
1. 파츠 1개 = 프리미티브 조합 후 `BufferGeometryUtils.mergeGeometries` 로 **머티리얼(팔레트 색)당 1메시**로 병합.
2. 라운드 프리미티브는 저폴리 고정: `radialSegments 8~12`, sphere `widthSegments ≤ 12`.

**수치 성능 예산(블래스터 draw call ≤56·파츠당 메시 ≤14·투사체 풀·씬 총 예산)의 단일 정본은 07 성능 예산 표다(결정문 23)** — 본 섹션은 참조만 하고 독자 수치를 두지 않는다.

**자유 변형과의 관계 (정본은 09)**: 슬라이더로 몸통·배럴을 주무르는 자유 변형(결정문 27)의 형태 범위는 09 §7 의 **토이 프로포션 봉투**(세장비·L/r·최소 단면 상한)가 **설계 시점 사전 보정**으로 보장한다 — 어떤 극단 morph 조합도 위 실루엣 판별식을 벗어날 수 없어야 하며, 최종 검증은 본 섹션 §5.3 체크리스트가 잡는다.

전작 패턴 재사용: 절차 생성 메시 + `MeshStandardMaterial` + `flatShading = true` (전작 `C:/ai-game-lab/src/visuals.ts` 의 머티리얼 헬퍼 패턴 — 66·149행 부근, 실재 확인됨). 텍스처 없이 색면(flat color) 분할로 장난감 사출 플라스틱 질감을 낸다.

### 1.2 색 — 고채도 원색 + 파스텔 (밀리터리 톤 전면 금지)

**규칙: 블래스터 1정에 최소 2색 이상.** 단색(특히 검정·군녹색 단색)은 실총처럼 보이므로 금지. Nerf 실물이 그러하듯 "주황 총구 팁"을 브랜드 시그니처로 채택한다.

```typescript
// src/game/palette.ts — 전 섹션 공용. 이 밖의 색을 쓰려면 이 파일에 추가부터.
export const TOY_PALETTE = {
  // 주역 원색 (몸통·파츠 메인)
  blasterBlue:   0x2f7fe8,
  blasterOrange: 0xff8a2b,  // 총구 팁 고정색 — 모든 블래스터 공통
  blasterYellow: 0xffd23f,
  blasterRed:    0xf05454,  // 채도 높지만 '피 빨강'(0x8b0000류 암적색) 아님
  blasterGreen:  0x4cd964,  // 라임 그린 — 군녹색(olive) 아님
  // 파스텔 (꾸미기·배경·타겟)
  pastelPink:    0xffb5c9,
  pastelMint:    0xa8e6cf,
  pastelSky:     0xbde0fe,
  pastelCream:   0xfff3d6,
  // 중립·다크
  toyGrayLight:  0xd9dde3,
  toyGrayDark:   0x6b7280,  // 어두운 회색 — secondary/accent 전용(칙칙한 몸통 방지)
  toyBlack:      0x1e2126,  // 검정 — 줄무늬·번개·타이어 그립 + 몸통 본체색 허용(2026-07-18 사용자 요청, canBePrimary)
} as const;

export type PaletteKey = keyof typeof TOY_PALETTE;

// 존 차등 규칙 — 저장 존 = 결정문 12 의 파츠 인스턴스별 paint 3존(primary/secondary/accent).
// tip 은 고정색이라 세이브 대상이 아니다. "실총처럼 보이는 어두운 단색 몸통"만 차단하고 표현 폭은 유지.
export type PaintZone = "primary" | "secondary" | "accent";
export const ZONE_ALLOWED: Record<PaintZone | "tip", "bright" | "any" | "orangeOnly"> = {
  primary: "bright",    // 밝은 색 + 검정(toyBlack) 허용. toyGrayDark 만 불가 (2026-07-18 canBePrimary — 사용자 요청으로 검정 예외)
  secondary: "any",
  accent: "any",        // 검정 번개 무늬·다크+네온 조합 가능
  tip: "orangeOnly",    // blasterOrange 고정 — 변경 UI 자체가 없음
};
```

이 존 차등 매핑을 파츠 지오메트리 존(ZoneId)별 색상 피커 필터로 구현하는 것은 **03 소관**이다(결정문 25 — "03 에 08 슬롯 차등 매핑 추가").

**저장 형식 (하드)**: 세이브 데이터·`ProjectileSpec`·꾸미기 상태는 hex 숫자가 아니라 **`PaletteKey` 문자열을 저장**하고 렌더 시점에 `TOY_PALETTE[key]` 로 해석한다. 팔레트 채도 튜닝(반드시 온다)이 세이브 마이그레이션 없이 기존 창작물 전체에 전파된다. **폴백 규칙 (결정문 11)**: 팔레트에서 키가 삭제·개명되어 세이브의 `PaletteKey` 를 해석할 수 없으면 존별 기본키(primary=`blasterBlue`, secondary/accent=`toyGrayLight`)로 **무소음 폴백**한다 — 에러 다이얼로그 없음.

**금지 색역(하드)**: 군녹색(olive/OD green 계열 0x556b2f±), 사막 탄색(tan/coyote), 건메탈·매트블랙 **단색 몸통**(primary 존), 암적색(핏빛). 자유 RGB 피커·색상환 금지 — 색 선택 UI 는 `TOY_PALETTE` 키 목록만 노출(결정문 11). 위장 패턴은 §2 G2 참조.

### 1.3 사운드 톤 — "뿅뿅·펑·휘리릭"

전작의 절차 SFX 패턴(`C:/ai-game-lab/src/game/audioKit.ts` — WebAudio 오실레이터 + 주파수 램프 + 필터, 실재 확인됨)을 그대로 재사용하되, **톤 규칙**을 뒤집는다. 전작 메모리의 "저역·노이즈 발소리 = 오류음처럼 들림" 교훈에 따라, 이번 게임은 저역·노이즈 중심 사운드를 금지 톤으로 규정한다. WebAudio 는 첫 탭에서 resume 하는 부팅 태스크가 M1 필수다(결정문 26).

| 이벤트 | 권장 톤 | 절차 SFX 레시피 예시 | 금지 |
|---|---|---|---|
| 발사 | "뿅!" / "퓽!" | square 파, 880→220Hz 하강 램프, 80ms | 실총 총성(노이즈 버스트+저역 붐), 반동음 |
| 명중 | "팅!" / "펑!" | sine 1200Hz + 배음, 짧은 팝 120ms | 살점음·비명·둔탁한 타격음 |
| 풍선 터짐 | "빵→휘리릭" | 노이즈 20ms 팝(고역만) + triangle 상승 글리사도 | 폭발 굉음(저역 롱테일) |
| 다트 채우기 (M2 — 결정문 21) | "찰칵-딸깍"(장난감 스프링) | square 짧은 클릭 2연타 + sine 스프링 바운스 | 실총 노리쇠 소리 샘플 |
| 콤보/별 획득 | "띠로링↑" | 도-미-솔 arpeggio, sine, 밝은 장조 | — |
| 코스 완주 팡파레 | "빠밤-띠로리링!" | 장조 arpeggio 2옥타브 상행 + 고역 반짝임 딜레이 | 웅장한 오케스트라풍(저역) |
| 실패/빗나감 | "뽀용~"(귀엽게) | sine 440→330Hz 짧은 하강, **단조 금지** | 부저음·경고음·야유 |

자유 변형 슬라이더의 드래그 사운드("뽀롱" 피치 슬라이드, 09 M1.5 — 정본은 09 §5)도 이 표의 톤 규칙(밝고 높게, 저역 금지)을 그대로 상속한다.

**전역 규칙 (2단 캡)**
- **스팸성 SFX** (발사·명중·다트 채우기·리스폰·슬라이더 드래그 — 초당 수 회 발생 가능): **≤ 400ms**. 경쾌함 유지.
- **보상 스팅어** (별 획득·코스 완주·신기록 — 회당 1회): **≤ 1.5s**, 밝은 장조 유지. 보상 순간까지 400ms 에 욱여넣어 "띡"이 되게 하지 않는다.
- 기본 주파수 300Hz 미만의 **단독 저역음 금지**(무섭게 들림).
- **저역 롱테일 잔향 금지**(전쟁터 느낌). 고역의 짧은 반짝임 딜레이/에코(≤ 300ms 테일)는 보상 스팅어에 한해 허용.

## 3. 어휘 사전 — 게임 내 텍스트 권장/금지 용어

UI 문자열·퀘스트 텍스트·파츠 설명 전부에 적용. **코드 식별자도 동일 규칙**(예: `damage` 변수명 금지 → `popPower`).

**정본은 아래 표다.** 기계 lint 배열은 표의 "기계 lint" 열이 O 인 항목만 담는다(표↔배열 불일치가 생기면 표가 이긴다). "총"·"적"·"사격" 같은 단독 1~2음절어는 정상 표현("총 3발", "적정", "사격장")과 충돌해 기계 차단이 오탐을 내므로 **사람 리뷰 항목**으로 분리한다 — §5 체크리스트에서 잡는다.

| 금지 (쓰지 마라) | 권장 (이걸 써라) | 코드 식별자 | 기계 lint |
|---|---|---|---|
| 총, 무기, 화기, 무기고 | 블래스터 / 보관 화면은 "진열장"(결정문 25) | `Blaster` | "무기"·"화기" O ("무기고"는 "무기" 부분매칭으로 자동 차단) / "총" 단독은 사람 리뷰 |
| 총알, 탄환, 탄약 | 다트 / 젤리볼 / 물감볼 | `Dart`, `JellyBall` | O |
| 사살, 킬, 처치 | 명중! / 팝! / 터뜨렸다! | `hit`, `pop` | O |
| 데미지, 대미지 | 팝 파워 (터뜨리는 힘) | `popPower` | O |
| 헤드샷 | 정중앙 명중! (퍼펙트 팝) | `perfectHit` | O |
| 사격, 발포 | 발사, 슛 (단, "사격장"·"사격 테스트" 복합어는 허용 — 스포츠 사격 뉘앙스. 사격 공간 한글명 = "사격장", 결정문 8) | `fire`, `shoot` (발사 의미로만) | "발포" O / "사격" 단독은 사람 리뷰 |
| 장전, 탄창 | 다트 채우기 / 다트 팩(결정문 2) / 다트 수 | `reload`, `dartPack`, `capacity`("다트 수" — ammo 금지, 결정문 25. 노출은 M2 부터, 결정문 21) | "탄창" O |

### 3.1 vocab lint 기계 명세 (오탐 0 을 목표로 한 매칭 규칙) — **금칙어 단일 정본 (결정문 25)**

스캔 대상: `src/**/*.ts` 의 **문자열 리터럴 + 식별자**. 자유 변형 슬라이더 라벨(`labelKo`·`minLabelKo`·`maxLabelKo`, 09 §1)도 문자열 리터럴이므로 **자동 포함**된다 — 명시 규정(결정문 25 연장, 09 §7.2). 구현은 정규식 + 토큰 분해 **~60줄 스크립트**로 M1 유지 — AST 파서(`ts.createSourceFile`) 도입 금지. `npm run verify` 게이트에 포함.

```typescript
// src/game/vocabRules.ts — 데이터만. lint 스크립트(scripts/vocabLint.mjs)가 import.
// 정본 = 설계 문서 §3 표. 이 배열은 표의 "기계 lint = O" 항목의 사본이다.
// 03 의 구 금칙어 목록은 여기로 병합 완료 — 다른 문서에 별도 목록을 두지 않는다 (결정문 25).
export const FORBIDDEN_KO: readonly string[] = [
  "총알", "탄환", "탄약", "사살", "헤드샷", "데미지", "대미지", "무기", "화기",
  "처치", "적군", "탄창", "발포", "폭파", "저격", "스나이퍼", "게임오버", "패배",
  "사망", "글록", "링스", "스콜피온", "베레타", "배레타",
];
export const ALLOWED_KO_COMPOUNDS: readonly string[] = [
  // 금지어를 부분 포함하지만 허용하는 복합어 (선허용 후매칭)
  // 현재 비어 있음 — "사격장"·"총구"는 애초에 기계 금지어("사격"·"총")가 아니므로 불필요.
];
export const FORBIDDEN_EN_TOKENS: readonly string[] = [
  // 식별자를 camelCase/snake_case 로 분해한 토큰과 "완전 일치"로만 비교
  "kill", "damage", "weapon", "ammo", "sniper", "gun", "bullet",
  "ak", "m16", "m4", "mp5", "glock", "awm", "mg3", "aug", "beretta", "skorpion", "scorpion",
];
```

**매칭 규칙 (하드)**
1. **한글**: 문자열 리터럴만 대상. `FORBIDDEN_KO` 를 부분 문자열로 매칭하되, 매칭 위치가 `ALLOWED_KO_COMPOUNDS` 항목 내부면 통과. 단독 1~2음절 리스크어("총", "적", "사격")는 기계 목록에 넣지 않는다(오탐원).
2. **영문 식별자**: camelCase·snake_case·숫자 경계로 토큰 분해 후 소문자화, `FORBIDDEN_EN_TOKENS` 와 **토큰 단위 완전 일치**. → `skill`·`skillTree` 통과, `killCount`·`dealDamage` 차단, `BREAK`/`SPEAK` 는 "ak" 에 안 걸림. `ammo` 차단 → 코드는 `capacity`(결정문 25).
3. **영문 문자열 리터럴**: `\b단어\b` 완전 단어 매칭(대소문자 무시).
4. 위반 1건이라도 있으면 `npm run verify` 실패. 예외가 필요하면 코드 주석 우회가 아니라 **이 문서 §3 표 개정 → vocabRules.ts 동기화** 순서로만.

## 4. 발사체·타겟·이펙트 안전 표현 카탈로그

다른 섹션(발사체 물리, 타겟 설계, 이펙트)은 **이 카탈로그 안에서만** 고른다. 카탈로그 확장은 본 섹션 개정으로만.

### 4.1 발사체 (전부 장난감 계열, 명중 결과는 전부 "무해한 변형")

**kind 표기의 정본은 04 다 (결정문 6)**: `ProjectileKind = "dart" | "gel" | "paint"`. 구 표기 jellyBall/paintBall 과 **suctionArrow(흡착 화살)·bubble(거품탄) 행은 삭제**했다 — kind 확장(예: 거품탄 재도입)은 04 kind 개정 + 본 카탈로그 개정을 함께 거쳐야 한다. 08 은 아래 안전 필드 제약만 소유한다.

```typescript
export interface ProjectileSpec {
  kind: ProjectileKind;     // 타입 정본 = 04 ("dart" | "gel" | "paint", 결정문 6)
  colorKey: PaletteKey;     // hex 직접 저장 금지 — 렌더 시점에 TOY_PALETTE[colorKey] 해석 (§1.2)
  hitEffect: HitEffectKind; // §4.3 카탈로그 값만
  trailEffect: "none" | "sparkle" | "rainbowRibbon";
}
```

| 발사체 | 형태(절차 메시) | 명중 표현 | 특징 부여 예시 | 마일스톤 |
|---|---|---|---|---|
| 다트 (dart) | 캡슐 + 끝에 반구 스펀지 팁(주황) | 팁이 눌리며 튕겨나감 | 기본. **포물선 탄도(04 정본, 결정문 19)**. M1 은 무한 탄창(결정문 21) | **M1** |
| 젤리볼 (gel) | 반투명 구, 살짝 출렁(scale 진동) | 말랑 찌그러졌다 복원 | 바운스 1회 가능 | M2 (배정 정본 = 04) |
| 물감볼 (paint) | 불투명 구 | 파스텔 물감 스플랫(§4.3 구현 명세) | 타겟에 색이 묻음 — 꾸미기 재미 연결 | **M2+ (강등)** |

> paint 를 M2+ 로 강등한 이유: "타겟에 묻는" 표현은 데칼/렌더투텍스처가 아니라도 수명 관리(리스폰 연동)가 얽혀 30분 증분으로 안 쪼개진다. **다른 섹션은 paintSplat 에 의존하는 설계를 M1 에 두지 마라.**
> 발사체 크기: 배럴 굵기 morph 가 `ShotProfile.projectileScale` 에 기여하는 합성식은 04 정본·09 §2.2 참조 — 08 은 "보이는 대로 맞는다"(시각·충돌 일치) 원칙만 상속시킨다.

### 4.2 타겟 (전부 무생물·장난감 — G3 상속)

M1 코스는 "풍선 마당" 1개(고정 과녁 3 + 풍선 6, 결정문 20) — 아래 카탈로그의 동심원 과녁·풍선 행이 M1 범위다.

| 타겟 | 명중 반응 | 리스폰 표현 |
|---|---|---|
| 동심원 과녁 | 링별 점수 팝업, 정중앙이면 별 이펙트 | 고정형 |
| 풍선 (단색/줄묶음) | "빵" + confetti, 끈이 팔랑 떨어짐 | 1.2초 후 아래에서 새 풍선이 부풀며 등장 |
| 깡통 피라미드 | 통통 튀며 무너짐(경쾌한 캉캉 소리) | 3초 후 뿅뿅 하나씩 다시 쌓임 |
| 나무 블록 | 쓰러짐(넘어질 뿐 부서지지 않음) | 스프링처럼 다시 벌떡 일어섬 |
| 드론 볼 (프로펠러 달린 공) | 뱅글 돌며 아래로 살포시 착지 | 다시 두둥실 떠오름 |
| 슬라임 젤리 (**무표정 젤리 큐브**) | 출렁 찌그러졌다 복원 — 물성만으로 귀여움을 낸다 | 즉시 복원 |
| 바람개비 | 명중 수만큼 빨리 돎(파괴 없음) | — |

**타겟 공통 룰**
- 파괴 잔해를 남기지 않는다. 모든 "터짐/쓰러짐"은 2초 안에 유쾌하게 복원된다.
- **눈·표정·신체 부위(팔·다리·입) 전면 기본 금지** — M1~M2 는 예외 없음. "무생물만 조준" 원칙에 회색지대(눈만 → 눈+입 → 팔 스텁 마스코트화)를 만들지 않는다. M3 이후 표정 도입을 원하면 열린 질문 2 를 먼저 결정하고, 도입 시 하드 3조항을 붙인다: ① 명중 전·중·후 항상 웃음 고정(놀람·아픔 표정 금지) ② 신체 부위 0개(눈 외 일절 금지) ③ §5 체크리스트에 "눈 달린 타겟" 항목으로 배우자 테스트 검증.

### 4.3 이펙트

| 이펙트 | 용도 | 사양 |
|---|---|---|
| confetti | 풍선·정중앙 명중 | 색종이 12~20장, TOY_PALETTE 랜덤, 0.8초 낙하 |
| starBurst | 퍼펙트 팝 | 노란 별 5개 방사, 0.5초 |

## 6. 시뮬레이션 피드백 · 경쟁 · 좌절 관리 — 정직한 수치, 긍정적인 말투

**원칙: 좌절 관리(감점 없음)와 정보 제거(비교 불가)를 혼동하지 않는다.** 전자는 지키고, 후자는 하지 않는다 — 파츠를 바꿔 끼고 형태를 주무르는 동기는 "성능이 달라지는 것"이고, 그게 핵심 기둥 3(창작→검증→개선)의 연료다.

- **수치 노출 범위 (결정문 3 으로 한정)**: 정직함의 원칙은 유지하되, 숫자가 보이는 곳은 **결과 화면 기록값**(코스 결과의 명중 수·사거리 기록·신기록 비교)뿐이다. **스탯 UI(파츠 카드·공방 슬라이더 패널)에는 숫자 노출 금지** — 별 5개(연속 채움, 정본 09 §4)와 ↑/↓ 화살표로만 표기한다. 스탯 정본은 02 의 6스탯+파생 1(결정문 3), 스탯→발사 변환은 04 ShotProfile(결정문 4).
- **트레이드오프는 명시**: 파츠 카드에 "팝 파워 ↑, 연사 ↓" 식으로 항상 표기(화살표·별 표기 — 숫자 아님). 모든 성능 파츠는 최소 1개 ↑ + 1개 ↓ — 트레이드오프가 없으면 조합 자체가 무의미해진다(02 파츠 로스터가 상속). morph 의 스탯 결합·체감 채널(연속 별·퍼짐 원 프리뷰)의 정본은 09 §4.
- **비교 축은 "내 이전 블래스터 vs 지금"으로 고정**: "사거리가 6m 늘었어!"(결과 화면 기록값이므로 수치 허용) — 개선을 스스로 확인하는 루프. 절대 평가("잘 만든 총" 등급 판정)는 하지 않는다.
- **프레임만 긍정 유지**: 낮은 결과도 사실은 보여주되 서술은 긍정으로 — "사거리는 짧지만 연사가 최고 속도예요!" (은폐가 아니라 말투 규칙이다).
- **점수는 항상 플러스**: 감점·마이너스·점수 잃음 없음. 빗나감 = 0점일 뿐. `score: number` 는 단조 증가만 허용.
- **별 등급으로 재도전 유도**: 코스 결과는 별 1~3개 — M1 은 명중 수 기준 간이 별점(결정문 20), 정식 점수판·콤보는 M2. 별 0개는 존재하지 않음 — 완주만 해도 별 1개 + "연습 완료!" (전작 튜토리얼 진행도 패턴처럼 달성형으로만 기록).
- **실패 연출 금지**: "FAIL/게임오버/패배" 화면 없음. 미달 시 "아깝다! 별 하나만 더 노려볼까?" + 원버튼 재시작(≤1초).
- **오류·복구 화면도 같은 톤 (결정문 26)**: 저장 실패 시 백업 키 복원 안내 화면·WebGL 컨텍스트 로스트 리로드 안내도 §3 어휘 + 긍정 프레임을 따른다 — 겁주는 경고 톤 금지, "잠깐만! 다시 불러올게" 톤. 세이브 로드의 무소음 복구 원칙(에러 다이얼로그 금지)은 09 §6·05 와 공유.
- **타인 비교는 옵트인 격리**: 기본 화면에 남(형제) 기록 비노출. 리더보드류를 만들 경우 "내 최고 기록 vs 오늘 기록"만 기본 표시, 타인 기록은 옵트인 화면으로 격리. 프로필별 저장 분리(구조 정본 = 05 SavedGame 의 프로필 키 구조, 결정문 13 — SAVE_VERSION 마이그레이션 패턴 상속)로 형제가 서로의 별을 덮어쓰지 않게 한다.
- **순수 꾸미기(색칠·코 모양 등 스탯 0 morph)에는 평가 없음**: 성능과 무관한 외형 창작은 점수·등급·판정 일절 없음. 성능 검증은 시뮬레이션의 몫이다.
