## 잔여 4건 재정 (RESOLVED)

### R1. 발사체 종류(ProjectileKind) = 3종 확정 — 04 를 3종으로 정렬
- **확정**: `dart`(M1 기본·포물선 g≈4·반경 0.025) / `gel`(젤리볼·M2) / `paint`(물감볼·M2+). **거품볼(foam) 폐기** — 재도입 필요 시 08 §4.1 공동 개정 절차로만.
- M1 의 유일 발사체는 `dart`. 04 §2.1·§12 의 4종 표에서 `foam` 행 삭제, `gel`·`paint` 마일스톤을 M2 로.
- 코드 union: `type ProjectileKind = 'dart' | 'gel' | 'paint'`.

### R2. 색칠 = M1 확정 — 07 을 M1 로 정렬
- **확정**: 3존 PaletteKey 채우기 + paint 스테이션은 **M1**(범위 정본 01·03·05·06 다수). 07 의 M3 배치는 오답 → `paint.ts`·paint 스테이션 M1 편입. M3 는 **패턴·스티커·진열장 확장**으로 재정의.

### R3. muzzle 부착 = 배럴 끝 앵커 승계 확정 — 02 를 정렬
- **확정**: `muzzle`(총구 액세서리)은 **바디 소켓이 아니다**. 배럴의 끝 앵커를 승계한다(09 §3.3). 02 §1 몸통별 소켓 표에서 muzzle 열 제거, "배럴 끝 승계" 각주. muzzle 파츠 자체는 M2+.
- M1 소켓 = body 가 제공하는 `barrel`·`sight`(+ 몸통별 차등). SocketId 문자열 = 슬롯명과 동일(`barrel`/`sight`), 03 의 `"mag"` 오기 → 사용 안 함(M1 에 mag 없음).

### R4. PaletteKey = 08 단일 레지스트리 확정 — 03 프리셋은 키 묶음으로
- **확정**: 색 키 정본 = 08 `TOY_PALETTE`(13키, 예 `blasterBlue`·`toyGrayLight`). 03 의 "6프리셋×6색 = 36키"는 **프리셋 = 정본 키의 이름있는 묶음**으로 재정의(신규 키 생성 금지). 저장값·폴백 기본키 모두 실존 13키 중 하나.
- primary 존 = 밝은 색 강제(08 존 차등). 팁(tip)은 주황 고정(가드레일).

## 주요 nit 일괄 재정 (구현 시 이 값을 쓴다)
- 온보딩 시작 파츠 = **팝콘 몸통 + 숏 스냅 배럴**(롱 레일·도트는 ★ 해금). 01 §1.3 정정. (재검 #5)
- 반동: 02 `computeStats` 는 **도(°) 단위** `recoilKickDeg`(0.2~2.5°)를 직접 산출(0..1 미사용). foam 삭제로 탄속 클램프는 dart 20~60 m/s. (재검 #6)
- 간이 별점 ★ 기준 = **명중 수**(04 소유). 01 §1.7 "전멸+시간" 표현 폐기. M1 풍선 마당: ★1=완주, ★2=명중 6+, ★3=명중 9(전 타겟). (재검 #12·#20)
- 홀드 연사 = **M1 이후**(06 정본). 07 §4 "누르면 연사"는 이월 표기. M1 = 클릭/탭 단발. (재검 #8)
- M1 색칠 인터랙션 = 화면 고정 존 버튼 3개(03 §3.3). 06 "3D 존 탭"은 M2. (재검 #10)
- 이름 짓기 모달 = **M2**. M1 자동명(예 "블래스터 1"). (재검 #14)
- verify 체인에 `check:vocab`(금칙어 lint, 08 §3.1) 포함. (재검 #11)
- 코스 모드 타입 = 04 `RangeModeId` 단일. 05 `CourseMode` 폐기, balloon_yard 모드 = `targetLane`. (재검 #9·유령)
