## 1.1 한 문장 컨셉

**"내가 조립하고, 슬라이더로 주무르고, 색칠한 — 세상에 없는 나만의 장난감 블래스터를 바로 옆 사격장에서 쏴보며 점점 더 멋진 총을 만들어가는 3D 블래스터 공방 게임."**

- 장르 표기: 창작 샌드박스 + 스포츠 사격 시뮬레이션 (전체이용가 감각)
- 핵심 판타지: "장난감 가게 사장님 아들이 뒷마당 공방에서 세상에 하나뿐인 블래스터를 만든다"
- 폭력 판타지가 아니라 **공작(工作) 판타지**다. 총은 무기가 아니라 '내가 만든 작품'이고, 사격장은 전투가 아니라 '작품 발표회'다.
- 차별점은 두 겹이다: ① 만든 것이 **성능으로 증명**되는 루프(§1.4 ②축), ② 프리셋 파츠 조합을 넘어 **파츠 자체의 형태를 슬라이더로 주무르는 자유 변형** — "프리셋 조합은 출발점이고, 파츠를 주무르는 것이 차별점"이다 (결정 27, **정본은 09**).

### 타깃 페르소나 (초3~6 눈높이)

| 항목 | 내용 |
|---|---|
| 대표 페르소나 | 초등 4학년 남아 "유누"(10세). 전작 야생 생존 게임 플레이어 |
| 좋아하는 것 | Nerf 블래스터 실물, 로블록스 창작류, 마인크래프트 건축, "내가 만든 거 아빠한테 자랑하기" |
| 읽기 수준 | 짧은 한국어 문장 OK. 긴 설명문·영어 UI는 스킵함 → **텍스트는 한 화면 1~2줄, 아이콘+색으로 말한다** (아이콘은 절차 SVG — 이모지 금지, 결정 25) |

| 성공 기준 | 첫 세션에서 "아빠 이것 봐, 내가 만든 총이야!"가 나오면 성공 |

## 1.2 코어 루프

코어 루프는 **조립·변형→사격→보상→해금 4박자**다. 도색은 공방의 병렬 스테이션으로 두고, 보관함은 작품 생성·복제·이름 변경·백업의 관리 스테이션으로 둔다. 공간은 06 StationId 5종(workshop/paint/range/pvp/collection)이 정본이며 사격 공간 한글명은 "사격장"이다. PVP는 보관함 작품을 AI 드론과 검증하는 선택 루프다.

```
            ┌──────────────────────────────────────────────────┐
            │                                                  │
            ▼                                                  │
   [1] 조립+변형 (공방)                                         │
   몸통(유일 필수 슬롯) + 슬롯 파츠(배럴·조준기 — M2: 다트 팩·    │
   개머리판·총구 액세서리) + 슬라이더 자유 변형(정본 09)          │
            │                                                  │
            ▼                                                  │
   [2] 시험발사 (사격장)                                        │
   과녁·풍선 코스에서 직접 사격 (M2: 캔타워·무빙 드론)            │
            │                                                  │
            ▼                                                  │
   [3] 점수/보상                                               │
   명중 수 → 별점(★1~3) + 신기록 + 관찰 힌트                    │
            │                                                  │
            ▼                                                  │
   [4] 해금 → "이번엔 숏 스냅을 두툼하게!" 재조립 ────────────────┘

   ‖ 병렬 스테이션: [도색대(paint)] — 공방에서 원할 때 진입 (루프 필수 아님)
   ‖ 파츠별 팔레트 색칠. 온보딩에서 1회 통과시켜 존재만 학습
```

- **한 바퀴 목표 시간: 2.5~4분.** (조립·변형 60~120초 → 사격 60~90초 → 결과 10초 → 해금·복귀 10초. 도색은 예산 밖)
- 루프의 엔진은 [3]→[4] 연결부다: 결과 화면에 **사실 기반 관찰 힌트** 1줄 + 남은 타겟 하이라이트를 노출해 재조립 동기를 만든다. "○○ 파츠면 됐을 텐데"식 반사실 처방은 하지 않는다 — 틀린 처방은 신뢰를 부수고, "정답 조합이 있다"는 메시지는 "코스별 내 조합" 철학과 모순이다. **처방이 아니라 관찰만 말한다** (구체 룰은 §1.7 #7).
- 자유 변형이 루프 [1]에 녹는 방식: 별도 "변형 모드" 없이 공방에서 파츠 탭 → 슬라이더가 나타나 즉시 주무른다. 슬라이더 패널의 "쏴보러 가기" 버튼이 [1]→[2]를 직결한다 (배치·UX 상세는 06·09 소유).
- 루프 이탈점 방지: 어느 화면에서든 원클릭으로 공방↔사격장 이동 (씬 전환 1초 이내, 같은 Three.js 씬 내 카메라 이동으로 구현).

### 루프를 지탱하는 최소 데이터 (스케치 — 정본: 세이브 루트 = 05, 조립 구조 = 02, morph = 09)

세이브 루트는 M1부터 확정한다(정본 = 05 `SavedGame` + 프로필 키 구조, 결정 13 — 아래는 루프 관점 스케치). 진행도(별·코스 오픈)는 **빌드가 아니라 프로필 레벨**에 둔다 — 아이가 블래스터를 뜯어고치거나 지워도 별이 사라지면 안 된다. localStorage 접두사는 `blaster_lab_` (결정 1).

```ts
// ── 스케치 — 정본은 02(Blaster 구조)·05(세이브 루트)·09(MorphState) ──
const SAVE_VERSION = 1;            // morph 포함해 v1 부터 (결정 13)

interface Blaster {                // 단일 조립 타입 (결정 12)
  id: string; name: string;        // 기본값 "블래스터 1호" (이름 짓기 UI는 M2)
  createdAt: number;
  parts: Partial<Record<SlotType, PartInstance>>;  // body 만 필수 (결정 16)
}
interface PartInstance {
  partId: PartId;                  // id 규약 = 03 (body_* 등, 결정 7)
  paint: PartPaint;                // 3존(primary/secondary/accent) PaletteKey (결정 11·12)
  morph: MorphState;               // 0..1 정규화 Record — 정본 09 (결정 12·27)
}
// 코인("스파크 코인", M2)·해금·코스 기록 필드는 05 세이브 루트가 선점 (결정 5·13)
```

스탯의 내부 정본은 **02의 6스탯(1~10) + 파생 1** 이다 (mobility·handling은 handling으로 통일, "멋" 스탯 없음 — 결정 3). 스탯→발사 변환은 **04 ShotProfile 계약·클램프**(탄속·퍼짐·연사 간격·반동)가 정본이다 (결정 4). 맨몸 몸통(파츠 0)의 기본 발사값은 `computeStats`가 정의한다 (결정 16).

**트레이드오프는 초등학생도 즉시 체감하게 과장 스케일**로 잡는다 — **모든 파츠는 얻는 것과 잃는 것이 한 개씩** 있어 "정답 조합"이 아닌 "코스별 내 조합"이 나오게 한다 (수치 정본은 02 §3 로스터 표). 자유 변형의 스탯 델타는 파츠 교체 델타보다 한 급 작다 — 파츠 선택이 큰 방향, morph는 미세 조정 (09 §4).

### 스탯 표시 원칙 — 숫자는 어른의 언어

내부 수치는 1~10 연속값(02)이지만, **화면에는 숫자를 쓰지 않는다** (결정 3 — 예외는 결과 화면 기록값뿐, 08 §6 범위 한정). 페르소나 정의("아이콘+색으로 말한다")대로:

- 스탯 = 절차 SVG 아이콘(이모지 금지 — 결정 25) + **별 5개** (06 방식. 자유 변형 드래그 중에는 연속 부분 채움 — 09 §4)
- 파츠 장착·해제·변형 순간 **변화분만 크게 팝업**: 별 단위 글리프(`정확 ▲` + 반 별)로 초록/빨강. `+1 정확` 같은 숫자 팝업은 금지 (결정 3, 09 §4)
- M1에 노출하는 스탯은 체감 가능한 것만 — capacity(한글 "다트 수")·reload는 무한 탄창인 M1에서 숨기고 M2부터 노출한다 ("체감 없는 스탯 금지" — 결정 21·25)

## 1.3 세션 플로우

### A. 첫 실행 3분 (온보딩 = 튜토리얼 없는 튜토리얼)

**첫 발사가 30초 안에 나온다.** 사격의 쾌감을 경험하기 전의 조립은 의미 없는 준비 작업이고, 스탯 별 변화는 비교 기준(맨몸 사격 경험)이 없으면 정보가 아니다. 그래서 순서를 뒤집는다: **먼저 쏘고 → 파츠를 달고 주무르고 → 다시 쏴서 개선을 몸으로 확인** — 온보딩 자체가 재미축 ②(창작→성능 증명)의 첫 증명이 된다. 필수 슬롯이 body뿐이라 "맨몸 첫 사격"이 성립한다 (결정 16).

| 시간 | 화면 | 아이가 보는 것 | 아이가 하는 것 |
|---|---|---|---|
| 0:00~0:30 | 사격장 | 눈앞에 풍선 3개 + 큰 화살표 "쏴보세요!" | **맨몸 몸통(파츠 0)으로 즉시 첫 사격** — 1인칭 사대에서 드래그 조준 + 클릭 발사 (결정 18). 약하고 흔들리지만 풍선은 **펑펑** 터짐 (파티클+효과음) |
| 0:30~1:30 | 공방 | 화살표가 작업대 지목 "파츠를 달면 세져!" — 파츠 카드(롱 레일 배럴·도트 사이트)가 반짝이고, 파츠를 고르면 슬라이더가 나타남 | 파츠 카드 클릭 → **딸깍** 부착 + 별 팝업. 이어서 **슬라이더로 몸통을 쭉 늘여봄 — 총이 손을 따라 변하는 첫 경험** (정본 09). 마음대로 붙였다 뗐다, 늘였다 줄였다 (실패 없음, 되돌리기 버튼 1개) |
| 1:30~2:00 | 도색대 | 색 팔레트 | 원하는 색 1~2개 칠함. 건너뛰기 가능 (병렬 스테이션의 존재만 학습) |
| 2:00~2:50 | 사격장 | 본코스 "풍선 마당" (고정 과녁 3 + 풍선 6 — 결정 20) | 재사격 — **아까보다 덜 흔들리고 잘 터짐**을 몸으로 확인 |
| 2:50~3:00 | 결과 | ★★☆ + "새 파츠 해금!" 카드 뒤집기 | 보상 수령 → 공방으로 자동 복귀, 루프 완성 |

설계 원칙: **3분 안에 코어 루프 [1]~[4]를 전부 1회전**시킨다. 설명 팝업 0개, 유도는 반짝임·화살표·사운드로만. 첫 코스는 실패 불가능하게 설계 — 간이 별점(명중 수 기준, 결정 20)의 ★1 하한을 절반 명중 수준으로 낮게 잡는다 (구체 문턱은 04·05 코스 테이블 소관).

|---|---|---|

| **② 성능 검증** "내가 만든 게 실제로 잘 나감" | 유능감·인과 확인 | 스탯이 사격에 **물리로 체감**됨(스탯→발사 계약 = 04 ShotProfile): 정확→탄퍼짐, 파워→대형 풍선 한방/두방·타겟 리액션, 연사→리듬 **[MVP]** · **형태가 곧 성능 — morph 스탯 결합 + 퍼짐 원 프리뷰 [MVP — 09 §4]** · 코스 별점·신기록 **[MVP]** · 결과 화면의 사실 기반 관찰 힌트+남은 타겟 하이라이트 **[MVP]** · power→캔이 날아가는 거리(페이크 아크 연출) [M2] · 같은 코스를 다른 조합으로 깨면 보너스 [M2] |
| **③ 수집·전시** "내 컬렉션 봐봐" | 수집욕·과시 | 공방 벽 **진열장**(3칸, 만든 블래스터가 실물 크기로 걸림) [M2] · 파츠 도감(실루엣→해금 시 컬러) [M2] · 진열장 앞 카메라 모드(스크린샷=아빠에게 자랑) [M2] · 코스 ★ 현황판 [M2 — M1은 코스 1개라 결과 화면 별 표시로 충분] |

①의 자유 변형과 ②의 성능 증명이 맞물린 것이 이 게임의 차별점이다. 꾸미기 게임은 많지만 **"주무른 형태가 성능으로 증명되는" 루프**가 창작을 반복하게 만든다. 같은 파츠 5종으로도 morph 조합이 다르면 다른 총이다 — 카탈로그 밖의 "세상에 없는 내 총"(09 §4). 스탯→물리 매핑은 과장되게 체감시키되(예: 고파워 타겟 1.5배 과장 연출) 강체 물리 시뮬 없이 정해진 아크를 재생하는 페이크로 한다.

M1은 ①+②만으로 성립한다. ③(수집·전시)은 만든 것이 2자루 이상 쌓이는 M2부터가 자연스러운 시작점이다.

### M1이 스스로 재조립을 강제하는 구조 (설계 의도)

타겟 2군의 요구 스탯이 다르다: 원거리 고정 과녁은 정확 게이트, 대형 풍선은 파워 게이트로 설계한다 (게이트 수치는 04·05 소관). 정확형 배럴(롱 레일)은 과녁을 잘 맞히지만 대형 풍선에 여러 발 들고, 파워 쪽으로 세팅하면 반대가 된다. ★2는 어느 쪽으로든 가능하되, **★3(시간 조건)은 한 세팅 편식으로는 빡빡해서 배럴 교체 + 슬라이더 미세 조정(굵기로 파워 보충, 길이로 정확 보충 — 09 §4) 실험이 자연스럽게 발생**한다. "배럴 교체 + 형태 주무르기"가 M1의 확실한 재조립 동기다.

### 없어도 되는 것 (명시적 연기)

- 코인("스파크 코인")·상점 경제 (M2 — 결정 5. 해금 대상이 5개 이상 되는 시점에 도입, 세이브 필드는 05가 선점)
- 탄창("다트 팩")·재장전·capacity/reload 스탯 노출 (M2 — 결정 21)
- 캔타워 (M2 — 히트 시 정해진 포물선 아크로 날아가는 **페이크 연출**, 속도만 power 비례. 강체 시뮬은 도입하지 않는다)
- 랜덤 섞기·프리셋 시드·변형 사운드 (M1.5 — 정본 09 §8. 랜덤 섞기가 M1.5 첫 항목)
- 3D 드래그 앤 드롭 조립 (M2 장식 — 클릭 스냅이 기본 조작으로 유지)
- 모바일 터치 (M2 — 데스크톱 우선 확정, 결정 22. 클릭 기반 조립이라 이식 비용 낮음)
- 패턴·스티커·이름 짓기·진열장·도감·코스 ★ 현황판·정식 점수판·콤보 (M2)
- 무빙 타겟·드론·복수 코스·코스 에디터 (M2~M3)
- 사운드 다양화 (M1은 발사음+명중음+딸깍 3종이면 충분, 전작 audioKit 패턴 재사용)
- 일시정지·이벤트 버스 계약 (M2 예약 — 결정 26, 발행 정의는 04)
- 랭킹·공유 등 온라인 요소 전부 (M3+, 옵션)
- 튜토리얼 시스템 (M1은 반짝임+화살표 하드코딩으로 대체)

**M1 완성 판정 문장: "아무 설명 없이 아이에게 넘겼을 때, 5분 안에 스스로 파츠를 바꿔 끼우고 슬라이더로 형태를 주무른 뒤 다시 쏘러 가면 통과."**

## 1.9 PVP AI 드론 팝 아레나 (2026-07-24 현재 구현)

- 상단 `PVP` 탭에서 보관함 블래스터를 고른다. 선택한 작품은 활성 블래스터가 되며, 경기 중에는 스냅샷을 사용한다.
- 무표정 구형 AI 드론 3기와 순서대로 1대1 라운드를 진행한다. 매 라운드 양측 체력은 10에서 다시 시작한다.
- 파워는 회당 팝 파워, 연사는 발사 간격, 정확은 퍼짐, 다루기는 조준 추종과 좌우 이동 속도에 연결한다. 수치 정본은 04 §13이다.
- 사람·동물·유혈·사망 표현은 없다. 체력 0은 라운드 종료·에너지 충전·재도전으로만 표현한다.
- PVP 진행 상태는 일시적이다. 기존 보관함·활성 블래스터만 사용하므로 `SAVE_VERSION` 변경이 없다.
